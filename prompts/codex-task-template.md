# Codex 单任务执行模板

在执行实现计划中的某个 Task 时，使用以下流程：

1. 复述本 Task 的输入接口和输出接口；
2. 检查前序实现是否与计划签名一致；
3. 写计划指定的失败测试；
4. 运行最小测试并确认失败原因正确；
5. 写最小实现；
6. 运行目标测试；
7. 运行相关测试集；
8. 运行 Ruff 和 mypy；
9. 检查是否有站点特有逻辑渗入 Core；
10. 检查日志和 fixture 是否包含敏感信息；
11. 提交一个原子 Git commit；
12. 输出修改摘要、测试结果和提交哈希。

不得跳过失败测试阶段。不得在一个 Task 中实现后续 Task 的功能。
